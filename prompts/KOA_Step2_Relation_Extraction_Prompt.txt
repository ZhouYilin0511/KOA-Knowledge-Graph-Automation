Abnormal Symptoms entities (including the mandatory pre-filtering step): Before extraction, mandatory systemic-symptom filtering must first be carried out independently. The filtering standard must be exactly the same as the systemic-symptom criteria in Rule ① Western medical diagnosis. Systemic symptoms—namely descriptions involving the five categories of mental status, diet/appetite, bowel and urination, sleep, and cold/heat—must never be used in this relation. Apart from systemic symptoms, only Symptoms entities describing abnormal manifestations may be used in this relation. Abnormal symptoms are any statements that explicitly describe the presence of symptoms, regardless of whether the symptom description contains contextual qualifiers such as activity or posture (for example, “活动时”, “行走时”, or “上下楼梯时”), or modifiers of degree, status, or scenario such as “可忍受”, “轻度”, “偶尔”, “轻微”, “稍”, “较甚”, “反复”, “时轻时重”, or “蹲起/上下楼困难”, or symptom descriptions with accompanying degree/relief modifiers such as “减轻”, “稍缓解”, or “缓解”. As long as the abnormal manifestation is explicitly present—for example, “膝关节疼痛可忍受”, “活动时膝关节疼痛可忍受”, “偶感膝关节活动受限”, “膝关节活动稍受限”, “膝关节疼痛缓解”, “休息后膝关节疼痛可稍缓解”, “膝关节疼痛时轻时重”, “反复膝关节疼痛”, “蹲起时膝关节困难”, “膝关节稍肿胀”, and “下蹲时膝关节疼痛减轻”—it should be regarded as an abnormal symptom. Systemic Symptoms entities (i.e., those involving the five categories of mental status, diet/appetite, bowel and urination, sleep, and cold/heat), as well as non-abnormal Symptoms entities (including those described as “无X”, negative, normal, or without obvious abnormality, such as “下肢无明显胀感”, “膝关节屈伸活动尚可”, and “无肢体水肿”), must never be used in this relation. All systemic symptoms must be filtered out first, and only then may the remaining non-systemic Symptoms entities satisfying the definition of abnormal symptoms establish the Western non-pharmacological therapy relation. It is strictly prohibited to merge or omit any eligible Symptoms entity because of semantic similarity or merely differences in modifiers (for example, “膝关节活动受限”, “膝关节活动受限明显”, “膝关节活动稍有受限”, and “行走时膝关节活动受限加重” must all be traversed in full and linked individually). 4. Abnormal Physical signs entities: Only Physical signs entities describing abnormal manifestations may be used in this relation. Abnormal physical signs are explicitly recorded abnormal findings, including any Physical signs entity that describes the presence of an abnormality, regardless of whether it carries positive markers such as “（+）” or “（±）”, or whether it is a VAS score-type sign, an activity-related sign (such as “膝关节活动度异常” or “膝关节异常活动”), or only a description of an abnormal state (such as “关节周围压痛”, “腧穴压痛”, “膝关节肿胀”, or “膝关节畸形”). All such independent abnormal specialized Physical signs entities must be included, and each entity must be handled separately. No eligible entity may be omitted because another similar or more specific entity exists. Tongue-and-pulse Physical signs entities, as well as non-abnormal, negative, or normal Physical signs entities (i.e., those described as negative such as “（-）”, normal, or without abnormality), must never be used in this relation. Only Physical signs entities explicitly described as positive (“（+）”, “（±）”) or as abnormal states (such as “膝关节肿胀”, “膝关节内翻畸形”, “浮髌试验（±）”, “髌骨研磨试验（+）”, “过伸过屈试验（+）”, or “膝关节活动度异常”) may be included.
III. Attribute extraction rules
For certain medication and therapy entities, the efficacy attribute must be extracted:
① Efficacy of medications: For example,
(TCM prescription / Chinese patent medicine / Western medicine, Efficacy of medications, efficacy description). The efficacy-description text must be extracted as the attribute value from the Context Anchor of the medication entity that shares the same Context Anchor. For all TCM prescription, Chinese patent medicine, and Western medicine entities, if the Context Anchor contains any description related to medication efficacy (such as “抗骨质疏松”, “消炎止痛”, or “通络止痛”), these descriptions must be mandatorily extracted as attribute values, and attribute triplets must be established. Special attention: 1. For each medication entity, efficacy descriptions must be extracted from all of its Context Anchors, and an attribute triplet must be established for each efficacy description extracted from each anchor. Every single Context Anchor of every Western medicine, TCM prescription, and Chinese patent medicine entity must be checked one by one without omission, in order to extract explicit efficacy descriptions such as “补钙”, “止痛”, “抗炎止痛”, “补液”, “消炎止痛”, and “解热镇痛”. If the same entity has multiple Context Anchors containing efficacy descriptions, a separate attribute triplet must be established for each distinct efficacy description. Omission of any efficacy description in any Context Anchor is strictly prohibited. 2. If a unified efficacy description appears in a Context Anchor (for example, when one treatment purpose corresponds to multiple medications), that efficacy description must be associated separately with all medication entities contained in that anchor, so that each medication entity is correctly matched to the corresponding efficacy description. 3. After extraction, the medication-entity list must be checked to confirm that all medication entities appearing in the Context Anchors have established attribute triplets whenever the anchor contains an explicit efficacy description. If a Context Anchor does not contain any explicit efficacy description, no attribute triplet should be established. 4. For TCM prescription entities, the Context Anchor frequently contains efficacy descriptions such as “补益肝肾” and “通络止痛”. These descriptions must be mandatorily extracted as Efficacy of medications attributes, even when the same descriptions are also used to establish TCM therapeutic method relations. For example, in “予独活寄生汤加减补益肝肾，通络止痛”, in addition to establishing the TCM therapeutic method relation, the TCM prescription entity “独活寄生汤加减” must also be assigned the Efficacy of medications attribute “补益肝肾，通络止痛”.
② Supplementing the efficacy of TCM prescriptions: This applies only to (TCM prescription, Efficacy of medications, efficacy description). Efficacy supplementation based on the “Therapeutic Method Prescription” relationship: For each TCM prescription entity, collect all the content associated with it through the “Therapeutic Method Prescription” relationship with TCM therapeutic method entities (i.e., the therapeutic method description text, including repeated mentions of the same therapeutic method, but retaining only the deduplicated content). Then, concatenate these therapeutic method descriptions using "、" to form a string such as “xxxx、xxxx”, and use this concatenated string as the Efficacy of medications attribute value for the TCM prescription, establishing an attribute triplet (TCM prescription: entity content, Efficacy of medications, concatenated string). This step must be strictly executed for every TCM prescription entity, regardless of whether it has a contextual efficacy description in the Context Anchor. After establishing the triplet, a special verification step must be performed to ensure that all TCM prescription entities associated through the “Therapeutic Method Prescription” relationship generate the corresponding Efficacy of medications attribute triplet. It must be ensured that every TCM prescription associated through this relationship receives this attribute, and no omission is allowed. For example, if the TCM prescription “身痛逐瘀汤” is associated with “活血化瘀” (activating blood and removing stasis) and “通络止痛” (relieving pain and unblocking meridians) through the “Therapeutic Method Prescription” relationship, then the attribute triplet would be generated as: (TCM prescription: 身痛逐瘀汤, Efficacy of medications, 活血化瘀、通络止痛). If it is only associated with “行气止痛” (moving qi and relieving pain), the attribute triplet would be generated as: (TCM prescription: 身痛逐瘀汤, Efficacy of medications, 行气止痛).
③ Efficacy of therapies: For example, (Western non-pharmacological therapy / Other TCM characteristic therapy / Acupuncture therapy, Efficacy of therapies, efficacy description). The efficacy-description text must be extracted as the attribute value from the Context Anchor of therapy entities that share the same Context Anchor. For all entities of type Western non-pharmacological therapy, Other TCM characteristic therapy, and Acupuncture therapy, if their Context Anchors contain descriptions related to therapeutic efficacy (such as “活血化瘀”, “疏经活络止痛”, preventive effects such as “避免关节僵硬”, or improvement-related effects such as “促进神经功能恢复”), these efficacy descriptions must be extracted mandatorily and without omission from each individual Context Anchor of each therapy entity, and an attribute triplet must be established for each one. Special attention: 1. Efficacy descriptions must be extracted for each therapy entity from all of its Context Anchors, and each efficacy description corresponding to each Context Anchor must independently generate one attribute triplet. It is strictly prohibited to merge efficacy descriptions from multiple anchors or to omit the efficacy description from any anchor. Every Context Anchor must be checked one by one, including but not limited to treatment-related, prevention-related, and improvement-related descriptions associated with the therapy. It is strictly prohibited to omit any efficacy description simply because the entity has multiple anchors. If the same entity has multiple Context Anchors containing efficacy descriptions, then a separate attribute triplet must be established for each different efficacy description, and omission of any efficacy description from any anchor is strictly prohibited. 2. If a Context Anchor contains a shared efficacy description (for example, when one therapeutic purpose corresponds to multiple therapies), that efficacy description must be associated separately with all therapy entities appearing in that anchor, so that each therapy entity is matched to the corresponding efficacy description. 3. After extraction, the list of therapy entities must be checked to confirm that all therapies appearing in Context Anchors have established attribute triplets whenever the anchor contains an explicit efficacy description, including preventive and improvement-related effects. If a Context Anchor does not contain any explicit efficacy description, then no attribute triplet should be established. 4. Every therapy entity type in the entity list—including Western non-pharmacological therapy, Other TCM characteristic therapy, and Acupuncture therapy—must have its Context Anchors checked one by one. This check is mandatory and must be performed without omission. If a Context Anchor contains any efficacy description, whether treatment-related, preventive, or improvement-related, then an Efficacy of therapies attribute triplet must be established. Omission of any therapy entity is strictly prohibited.
IV. Output requirements
1. All triplets must be output strictly in the following formats:
Relation triplet: (Entity Type: Entity, Relation Type, Entity Type: Entity) Note: the order of the head entity and tail entity must strictly follow the fixed order defined for each relation type and must not be reversed or exchanged. The entity must be the pure entity content with the Context Anchor removed, i.e., only the Entity part extracted from the entity list, without including “::” or any contextual description following it.
Attribute triplet: (Entity Type: Entity, Attribute Type, Attribute Value) Likewise, the entity must be the pure entity content after removing the Context Anchor, and must not contain any Context Anchor information.
2. Output only one triplet per line, without any explanatory text.
3. It must be ensured that all possible relations and attributes are extracted without omission. In particular, for attribute triplets, extraction must strictly follow the attribute-extraction rules. For the same entity (including its Entity Type) under the same Attribute Type, all different Attribute Values must be retained in full. It is strictly prohibited to delete or merge any attribute triplet merely because the attribute values differ.
4. Priority should be given to matching entity pairs based on the Context Anchor. For entities without a Context Anchor, matching should be performed according to semantic relations.
5. Strict deduplication must be performed when outputting triplets. For relation triplets in which the head entity (including Entity Type), relation, and tail entity (including Entity Type) are completely identical, or for attribute triplets in which the entity (including Entity Type), Attribute Type, and Attribute Value are completely identical, only one instance should be retained, and repeated output is prohibited. Special attention: deduplication applies only when the head entity (including type), relation, and tail entity (including type) are all exactly the same, and “exactly the same” must be determined by strict string matching of all characters, without any semantic-similarity processing. If entity descriptions differ in wording and are not completely identical (for example, “膝关节无畸形” vs. “膝关节未见明显畸形”, “膝关节半月板回声不均匀” vs. “膝关节外侧半月板回声不均匀”, or “膝关节内侧半月板撕裂” vs. “内侧半月板撕裂”), they must be treated as different head or tail entities even if their meanings are similar, and deduplication must not be performed; the corresponding triplets must be output separately. After the full triplet list has been generated, a deduplication step must be executed to ensure, through string-based comparison of the output, that no duplicate triplets remain. Only completely identical triplets may be removed. For attribute triplets in which the same entity has multiple different attribute values, all of them must be retained, and no omission due to deduplication is allowed. This process is independent of the Context Anchor. Deduplication is a mandatory step before output and must not be omitted.
6. Attribute-value punctuation consistency check: After generating the attribute triplets, but before output, punctuation consistency processing must be performed on the Attribute Value field. It must be confirmed that, in all attribute triplets, the separators between coordinated phrases in the Attribute Value have been uniformly normalized to the Chinese enumeration comma “、”, so that duplicate attribute triplets caused only by punctuation differences can be eliminated through standardization. Note: this processing applies only to the Attribute Value field, so as to ensure string consistency during deduplication; such standardization must not affect the entity content or any other part of the triplet.
7. It must be ensured that the Entity content of all triplets, excluding any Context Anchor, is used for deduplication comparison.
8. Before performing any relation or attribute extraction, a global systemic-symptom filtering step must first be applied to all Symptoms entities, so as to generate a global non-systemic Symptoms entity list. It must be ensured that the relations of Western medical diagnosis and TCM disease differentiation strictly use this list. For relations such as Western medicine treatment that require abnormal symptoms, an additional independent abnormal-symptom screening step must then be carried out on top of this list.
9. Fallback filtering rule for entity-type errors: After all triplet lists (both relation and attribute triplets) have been generated and deduplicated, but before final output, a final entity-type rationality check must be performed on all triplets. If any entity appearing in any triplet (relation or attribute triplet) shows an obvious mismatch between its Entity Type and its Entity content, then the original entity-type annotation should be regarded as erroneous. For example, if the entity content is “三黄膏”, which is a common Chinese patent medicine ointment, but its type is labeled as TCM prescription; or if the entity content is “骨痛灵汀（骨痛灵酊）”, “通络祛痛膏”, “伤湿止痛膏”, or “跌打消炎散”, all of which are clearly Chinese patent medicine, but the type is labeled as Western medicine; or if the entity content is “洛索洛芬贴膏”, “鲑降钙素鼻喷雾剂”, or “维生素D2注射液”, all of which are clearly Western medicine, but the type is labeled as Chinese patent medicine; or if the entity content is “关节错缝术” or “手指点穴”, which are clearly Other TCM characteristic therapy, but the type is labeled as Western non-pharmacological therapy, and so on—then the entity-type annotation must be judged as incorrect. For any triplet involving such an incorrectly labeled entity, the entire triplet must be directly ignored and removed from the final output list. It must not be included in the final output under any circumstances.
